先頭に32バイトのヘッダがあります。
そのあとに可変長データが入ります。
データフィールドの最大サイズは1024バイトです。
データフォーマットは OperationCode毎に定義します。
まくおヘッダの各フィールドの意味は以下の通りです。

- ProtocolVer
 まくおプロトコルのバージョン番号。
 バージョン番号の異なるプロトコルは処理しません。

- Operation
 オペレーションコード
  MAKUO_OP_PING: 
  MAKUO_OP_EXIT:
  MAKUO_OP_FILE:
  MAKUO_OP_MD5 :

- RequestID
 コマンドを発行する度にインクリメントする

- status
 現在の状態

- old status
 以前の状態

- data size
 データフィールドのバイト数

- flags
 bit
  0: ACKなら1
  1: データ部が暗号化されている場合は1
  2:

- Sequence Number


「データフォーマット」

[ping]
プログラム起動時に送出するパケット。
このパケットを受け取ったノードは、送信ノードをメンバリストに追加する。
さらに、ユニキャストでPONGを返す。
